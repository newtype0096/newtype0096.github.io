🛠️ ‘wpfui’와의 첫 만남

최근 토이 프로젝트를 하면서 UI 라이브러리를 찾다가 ‘wpfui‘라는 것을 알게 됐다.
WPF에서 Fluent Design System 스타일의 컨트롤을 쓸 수 있게 해주는 아주 좋은 라이브러리다.

디자인도 깔끔하고 기능도 괜찮아 보여서 바로 사용 시작!
한참 잘 쓰다가…

🤨 SplitButton의 미묘한 불편함 발견

문제의 컨트롤은 SplitButton.
토글 버튼을 클릭하면 Flyout(팝업 메뉴)이 나와야하는데, 간간히 메인 버튼의 커맨드가 실행됐다.

“내가 뭔가 잘못 썼나?”
처음엔 내 코드 문제인 줄 알고 디버깅했는데, 아무리 봐도 라이브러리 자체가 이상한 것 같았다.
그래서 wpfui 코드를 직접 까보니…

🚨 버튼의 특정 부분을 클릭해야만 Flyout이 열리는 상황이었다!

즉, 버튼의 텍스트 영역(아래 빨간 부분) 을 클릭해야만 Flyout이 정상적으로 열리는 거다.

📌 문제 화면:

와… 이건 좀 아니다 싶었다.
일반적인 UI 기대 동작과도 다르고, 실제로 써보면 너무 불편하다.

✍️ 스타일 수정, 그리고 속이 시원해짐

결국 직접 스타일을 수정하기로 했다.
버튼의 특정 부분에서만 트리거되던 걸 버튼 어디를 클릭하든 Flyout이 열리게 변경!

📌 수정 후 화면:

이제야 좀 자연스러운(?)동작이 됐다.

🚀 Pull Request 도전! 오픈소스 기여 GO

수정하면서 문득 “이거 나만 불편한 게 아닐 텐데?” 싶어서,
혹시 같은 문제를 겪은 사람이 있는지 wpfui의 GitHub Issue를 확인해 봤다.

…역시나! 🤣
이미 동일한 문제가 제기된 이슈가 있었고, 해결되지 않고 있었다.

이럴 때가 바로 Pull Request(Pull 요청, PR) 때다!
바로 해당 Issue를 참조하면서 PR을 작성하고 올렸다.

📌 PR 화면:

⏳ 이제 Merge를 기다리며…

PR을 보냈으니 이제 Maintainer의 리뷰를 기다려야 한다.
그런데 보니까… 이미 PR이 잔뜩 쌓여있음. 😅
언제 Merge될지 모르겠지만, 꼭 반영돼서 이런 큰 오픈소스 프로젝트의 Contributors 목록에 내 아이디가 박히면 좋겠다!

🔥 Merge될 때까지 기다려보자!
